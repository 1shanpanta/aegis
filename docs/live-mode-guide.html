00
Decide

Decide before you start.

~5 min

The simulator path is shipped, tested, and submission-ready. Live mode is a strict upgrade in pitch power but adds five external moving parts: Lace, faucet, indexer, proof server, and the preprod node. If any one stalls, you burn submission-critical hours.

Time budget

Final Devpost submission is due Sunday May 17, 11:45 AM ET. If you have less than 6 hours before submission, stop reading this guide and finish Path A first (record video, deploy web/ to Vercel, fill the Devpost form). Come back here after.

What live mode actually proves

  • The contract really compiles and deploys to a Midnight node, not just an in-browser simulator.
  • A user with Lace can sign a real circuit-proof tx that updates on-chain state.
  • Counterparties on the chain can read the public ledger fields (state, txCount, lastTxCommitment) but cannot recover the witnesses.

What live mode does not change

  • The Compact contract source. Same aegis.compact compiled the same way.
  • The 10 contract tests. They keep running against the simulator.
  • The marketing site, the hero, the comparison, the video section.
06
Wallet

Install Lace Beta wallet.

~5 min

Lace is the official Midnight wallet, distributed as a Chrome extension. The stable Lace release does not support Midnight yet, you need the Beta channel which bundles the mnLace dApp connector.

  1. Open Chrome (or any Chromium browser). Firefox is not officially supported for Midnight yet.
  2. Go to lace.io and download the Beta. Look for the small “try Lace Beta” link, not the stable button.
  3. Install, create a new wallet, write the recovery phrase somewhere safe (you will throw this wallet away later, but Lace will not let you skip the step).
  4. Open the extension, click the network selector, choose Preprod. If you don’t see Preprod, your Lace is the stable channel, not Beta.
  5. Find your addresses (see the click-by-click below) and copy both Unshielded (mn_addr...) and Shielded (mn_shield-addr...). The faucet needs the unshielded one.

Where the addresses live inside Lace

The first time you open Lace after switching to Preprod it lands on the Cardano view, not Midnight. There’s a network/asset switcher at the top of the wallet panel. Click your way through this order:

  1. Click the Lace extension icon in your toolbar to open the wallet panel.
  2. At the top of the panel, find the asset / network switcher (looks like a small chip with the current network name, sometimes labelled Cardano). Click it.
  3. Pick Midnight (or Midnight Preprod) from the dropdown. The whole wallet view re-renders for the Midnight account.
  4. Click the Receive button (right next to Send on the main wallet screen).
  5. Lace shows the three Midnight addresses stacked vertically, each with a label and a copy icon:
    • Unshielded — starts with mn_addr.... Use this one for the faucet.
    • Shielded — starts with mn_shield-addr.... Used for private Zswap transfers.
    • DUST — starts with mn_dust.... You don’t paste this anywhere; DUST regenerates automatically from your tNIGHT balance, you can’t send it.
  6. Click the copy icon next to Unshielded. Lace flashes a checkmark when copied.
  7. Optional: also copy the Shielded address for later (Aegis itself doesn’t need it for the faucet, but you’ll use it for any subsequent private transfers).
If you don’t see a Receive button

You’re still on the Cardano view. Re-check the asset/network switcher at the top and confirm it reads Midnight (not Cardano, not Cardano Preprod). Cardano addresses start with addr1... and won’t work on the Midnight faucet.

Programmatic alternative

Inside the dApp once Lace is connected, you can read the addresses without opening the extension UI: 

const lace = (window as any).midnight.mnLace;
const api  = await lace.enable();
const addresses = await api.getShieldedAddresses();
// addresses = { unshielded, shielded, dust }

That snippet is how the Aegis live engine pulls the addresses on the user’s behalf.

Why three addresses

Lace exposes three addresses derived from one seed: unshielded (public balance), shielded (Zswap, hides amounts and recipients), and a DUST address (non-transferable fee token). The faucet drops tNIGHT into your unshielded address. DUST regenerates automatically from your tNIGHT balance over time, like a battery from a power bank.

07
Faucet

Fund from the preprod faucet.

~5 min

Midnight uses a battery model. You hold tNIGHT (test NIGHT), it generates tDUST over time, and tDUST is what pays your transaction fees. You can’t transfer DUST. The first faucet drop will sit at zero DUST for 2-3 minutes while the battery charges, then settle.

  1. Open faucet.preprod.midnight.network.
  2. Paste your unshielded address (the mn_addr... one from Lace).
  3. Solve the CAPTCHA, click Request tNIGHT.
  4. Switch to Lace, refresh the balance. tNIGHT should appear within ~30 seconds. Wait ~2 minutes more for tDUST to start regenerating.
Sanity check

You’re ready when Lace shows both a non-zero tNIGHT balance AND a small non-zero tDUST balance. ~0.5 tDUST is enough for several Aegis transactions.

Rate limit

The faucet rate-limits per address per 24h. If you blew through your DUST and need more, either wait for the battery to regen or create a fresh Lace account.

08
Proof server

Run a local proof server in Docker.

~15 min (first run downloads ZK params)

Circuit proofs are expensive. Midnight runs them outside the wallet so the extension stays responsive. The official image bundles every prover key and ZK parameter file Midnight needs. 

# Pull and run the proof server, exposing port 6300 to the browser.
docker run -d \
  --name aegis-proof-server \
  -p 6300:6300 \
  -e PORT=6300 \
  midnightntwrk/proof-server:8.0.3

First run downloads ~500 MB of ZK params (cached after that). Tail the logs to confirm:

docker logs -f aegis-proof-server

You should see something like:

[INFO] proof-server listening on 0.0.0.0:6300
[INFO] ready

Smoke test from the host:

curl -s -o /dev/null -w "%{http_code}\n" http://localhost:6300/health
# expect: 200
First-run latency

The first circuit you prove after starting the server warms up an ~80 MB working set. Expect a 4-8 second pause on the very first createAllowance tx. Subsequent proofs are sub-second on a modern Mac.

Create web/src/lib/live-providers.ts

// SPDX-License-Identifier: Apache-2.0
// Provider stack for the live Midnight preprod network.

import { LevelPrivateStateProvider } from "@midnight-ntwrk/midnight-js-level-private-state-provider";
import { indexerPublicDataProvider } from "@midnight-ntwrk/midnight-js-indexer-public-data-provider";
import { httpClientProofProvider } from "@midnight-ntwrk/midnight-js-http-client-proof-provider";
import { FetchZkConfigProvider } from "@midnight-ntwrk/midnight-js-fetch-zk-config-provider";
import { setNetworkId, NetworkId } from "@midnight-ntwrk/midnight-js-network-id";
import type { MidnightProviders } from "@midnight-ntwrk/midnight-js-types";

const PREPROD = {
  indexer:   "https://indexer.preprod.midnight.network/api/v1/graphql",
  indexerWS: "wss://indexer.preprod.midnight.network/api/v1/graphql/ws",
  node:      "https://rpc.preprod.midnight.network",
  proofUri:  "http://localhost:6300",
};

export const PRIVATE_STATE_ID = "aegisPrivateState" as const;

export async function buildLiveProviders(): Promise<MidnightProviders> {
  setNetworkId("Preprod" as NetworkId);

  const lace = (window as any).midnight?.mnLace;
  if (!lace) throw new Error("Lace wallet not detected. Install Lace Beta.");

  const walletApi = await lace.enable();
  const uriConfig = await lace.serviceUriConfig();

  return {
    privateStateProvider: new LevelPrivateStateProvider({ databaseName: "aegis-private-state" }),
    publicDataProvider:   indexerPublicDataProvider(
                            uriConfig.indexerUri   ?? PREPROD.indexer,
                            uriConfig.indexerWsUri ?? PREPROD.indexerWS
                          ),
    zkConfigProvider:     new FetchZkConfigProvider(window.location.origin),
    proofProvider:        httpClientProofProvider(PREPROD.proofUri),
    walletProvider:       walletApi,
    midnightProvider:     walletApi,
  };
}

Create web/src/lib/live-engine.ts

// SPDX-License-Identifier: Apache-2.0
import { deployContract, findDeployedContract } from "@midnight-ntwrk/midnight-js-contracts";
import { combineLatest, from, type Observable } from "rxjs";
import { map } from "rxjs/operators";
import {
  CompiledAegisContractContract,
  createAegisPrivateState,
  ledger,
  type AegisPrivateState,
  type Ledger,
} from "@aegis/contract";
import { buildLiveProviders, PRIVATE_STATE_ID } from "./live-providers";

export async function deployAegisOnChain(initial: AegisPrivateState) {
  const providers = await buildLiveProviders();
  const deployed = await deployContract(providers, {
    contract:            CompiledAegisContractContract,
    privateStateId:      PRIVATE_STATE_ID,
    initialPrivateState: initial,
  });
  return { providers, deployed };
}

export async function joinAegisOnChain(address: string, initial: AegisPrivateState) {
  const providers = await buildLiveProviders();
  const deployed = await findDeployedContract(providers, {
    contractAddress:     address,
    contract:            CompiledAegisContractContract,
    privateStateId:      PRIVATE_STATE_ID,
    initialPrivateState: initial,
  });
  return { providers, deployed };
}

export function aegisState$(providers: any, address: string) {
  return combineLatest([
    providers.publicDataProvider
      .contractStateObservable(address, { type: "latest" })
      .pipe(map((s: any) => ledger(s.data))),
    from(providers.privateStateProvider.get(PRIVATE_STATE_ID))
      .pipe(map((p) => p as AegisPrivateState)),
  ]).pipe(map(([pub, priv]) => ({ public: pub as Ledger, private: priv })));
}

Update web/src/lib/aegis-engine.ts

Add a mode field to the engine. When mode === "live", route createAllowance, spend, and revoke calls through deployed.callTx.* instead of the simulator. Subscribe the React store to aegisState$ instead of the simulator's snapshot.

// Pseudocode for the routing layer inside the engine.

if (mode === "live") {
  await deployed.callTx.createAllowance(principalPk, agentPk);
} else {
  simulator.createAllowance();
}

if (mode === "live") {
  await deployed.callTx.executeAgentTx(amount, counterparty.address, nonce);
} else {
  simulator.executeAgentTx(amount, counterparty.address, nonce);
}
State subscription

The simulator returns ledger state synchronously after each call. Live mode is async, the tx hits the mempool, then a block, then the indexer. Wrap UI updates in optimistic state plus a pending badge that clears once aegisState$ emits a matching txCount.

Witness state in the LevelDB store

The simulator holds private state in memory. Live mode persists it in LevelPrivateStateProvider, which is an IndexedDB-backed key/value store keyed by your PRIVATE_STATE_ID ("aegisPrivateState"). The principal’s machine and the agent’s machine each have their own LevelDB — private state never crosses the network.

Toggle

Add the Simulator ↔ Live toggle.

~30 min

Judges without Lace need the simulator to keep working. Hide live behind a toggle that persists in localStorage and reflects in the URL (?mode=live). 

// web/src/lib/use-mode.ts
import { useEffect, useState } from "react";

export type Mode = "simulator" | "live";

export function useMode(): [Mode, (m: Mode) => void] {
  const [mode, setMode] = useState<Mode>(() => {
    if (typeof window === "undefined") return "simulator";
    const qs = new URLSearchParams(window.location.search);
    if (qs.get("mode") === "live") return "live";
    return (localStorage.getItem("aegis:mode") as Mode) ?? "simulator";
  });

  useEffect(() => {
    localStorage.setItem("aegis:mode", mode);
    const qs = new URLSearchParams(window.location.search);
    qs.set("mode", mode);
    window.history.replaceState({}, "", "?" + qs.toString());
  }, [mode]);

  return [mode, setMode];
}

UI: add a small pill toggle to the top bar in Hero.tsx and the demo section.

<div className="flex items-center gap-1 rounded-full border border-ink-700 p-0.5">

  </button>

</div>

11
Deploy

Deploy Aegis once, pin the address.

~10 min (after step 9 ships)

Each new deployContract call creates a fresh Aegis instance on chain. For the public demo we want one canonical instance that judges can join, so the demo doesn’t require everyone to spin up their own.

  1. Run the dev server with ?mode=live.
  2. Connect Lace, confirm you’re on Preprod.
  3. Click Create allowance. Lace pops up to sign; the proof server takes a few seconds.
  4. Watch the dev tools network tab. When the tx finalizes, the indexer subscription emits a new ledger state with state: ACTIVE.
  5. Copy the contract address from the UI (the engine should surface deployed.deployTxData.public.contractAddress).
  6. Set it as the default in web/src/lib/live-engine.ts: 
    export const AEGIS_PREPROD_ADDRESS = "0200deadbeef...";
  7. When the live page loads with no contract address in the URL, default to joinAegisOnChain(AEGIS_PREPROD_ADDRESS, ...) instead of a fresh deploy.
Witness fork warning

The pinned contract’s witnesses (cap, whitelist) are owned by whoever deployed it. Anyone who joins it as the agent must agree on those witness values, or their proofs reject. For the demo, the deployer’s spend cap should be generous (say $100 / week) and the whitelist should include the canonical demo counterparties already hardcoded.

Smoke test

Test the full loop.

~30 min

The end-to-end flow you’re proving:

StageWhat happensExpected
Open page ?mode=live, Lace detected, engine builds providers, indexer subscription opens. 200 OK
Connect wallet Lace popup → user clicks Connect. accepted
Create allowance Witnesses initialized in LevelDB, Compact circuit produces a proof at the local proof server, tx broadcasts to preprod. pending → finalized
Indexer catches up Public ledger state updates: state = ACTIVE, principalPk, agentPk. visible
Execute agent tx Agent click → circuit proves amount + spend ≤ limit & counterparty whitelisted → commitment emitted. accepted
Over-cap attack Drain attempt; circuit fails assert. circuit reject
Off-whitelist attack Spend to Tornado address; circuit fails assert. circuit reject
Revoke Principal clicks revoke; state flips to REVOKED on the public ledger. visible

For each step:

  1. Watch Lace for prompts. Each tx asks for confirmation.
  2. Watch the Docker logs: docker logs -f aegis-proof-server. You should see a proof request per circuit call.
  3. Watch the dev tools Network tab. The GraphQL subscription should push state updates.
  4. Confirm the public-pane in the UI shows the new txCount and lastTxCommitment within ~10 seconds of finalization.

99
Troubleshooting

When something breaks.

—

window.midnight.mnLace is undefined

  • Lace not installed, or stable Lace (not Beta). Reinstall the Beta channel.
  • Lace is asleep; click the extension icon to wake it.
  • You’re on Firefox or Safari. Use Chromium.

Proof server times out

  • Verify docker ps shows the container running.
  • curl -v http://localhost:6300/health should return 200.
  • If first run, ZK params still downloading (~500 MB). Watch docker logs.
  • If your browser CORS blocks localhost:6300, run dev with HTTPS or proxy through Vite.

Indexer subscription disconnects

  • Preprod indexer occasionally hangs. The provider will reconnect; UI should retry the subscription on disconnect.
  • Hard-refresh the page to rebuild the providers.

Tx rejected: insufficient DUST

  • Wait 60 seconds for the battery to regen.
  • If still empty, faucet a fresh address.

Tx rejected: assertion failed at the proof server

  • One of your witnesses doesn’t match. The most common case: the agent’s LevelDB has different allowanceLimit or whitelist than the principal’s. Reset both LevelDBs with the same initial state.
  • Check the proof server logs for the specific failed assert message.

Browser bundle exceeds 10MB after wiring

  • Midnight ledger WASM is 10MB. This is expected. Vite already chunks it separately, gzipped is ~4.7MB.
  • For Vercel deploy: ensure your output directory is set to web/dist.

If everything just refuses to work

The simulator submission is already enough to win. Ship Path A and treat live mode as a v1.1 milestone. The 10 passing contract tests prove the privacy primitive holds, and the in-browser demo proves the contract code executes correctly. Live mode is upside, not floor.
